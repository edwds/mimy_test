# MIMY UX Writing Guide (KO)

## 1) Tone & Manner
### 핵심 톤
- 개인화: "나만의 입맛/취향"을 분명히 드러낸다.
- 공감: "실망하셨나요?"처럼 사용자의 경험을 먼저 인정한다.
- 확신: 추천/분석/랭킹은 자신감 있게 제시한다.
- 미식 감성: 입맛, 풍미, 미식가 등 브랜드 어휘를 유지한다.
- 동행: 혼자가 아닌 "함께하는 발견"을 강조한다.

### 말투 규칙
- 기본 문체: 부드러운 해요체.
- 기본 호칭: {{name}}님 / 당신 (상황에 맞게 선택).
- 제안: "해보세요"를 기본으로 사용.
- 요청/필수: "해주세요"로 명확히.
- 확인/경고: "~할까요?" 또는 "~하시겠어요?"로 완화.
- 감탄/과장 표현은 최소화하고, 대신 확신 있는 정보로 설득.

### 문장 구성
- 질문은 한 문장만 사용하고, 이후에는 바로 행동을 안내한다.
- 한 문장은 짧게, 필요 시 줄바꿈(<br/>)로 리듬을 준다.
- 버튼/라벨은 동사형(기록하기, 저장, 닫기)으로 통일.

### 금지/주의
- 의미 없이 길게 쓰지 않는다.
- 같은 기능은 같은 단어로 말한다.
- 불필요한 존칭/격식체("~하시겠습니까?")는 지양.


## 2) UX Writing Patterns
### CTA
- 형식: 동사 + 결과
- 예: "기록하기", "추천받기", "랭킹 정리하기"

### Empty State
- 형식: 상황 인정 + 다음 행동
- 예: "아직 기록이 없어요. 첫 기록을 남겨보세요."

### Error/Alert
- 형식: 문제 안내 + 해결 방법
- 예: "네트워크 오류가 발생했어요. 다시 시도해 주세요."

### Confirm
- 형식: 짧고 부드럽게
- 예: "정말 삭제할까요?", "이 동네로 등록할까요?"


## 3) Terminology (KO 기준)
- 맛집: 개인화된 추천/랭킹 맥락의 장소
- 식당: 중립적 장소 표현
- 기록: 방문/리뷰/랭킹을 포괄하는 핵심 행동
- 랭킹: 사용자 기준의 순위 체계
- 추천: 개인화된 매칭 결과
- 가고 싶어요: 위시리스트 표현(띄어쓰기 유지)
- 콘텐츠: 컨텐츠 → 콘텐츠로 통일


## 4) Localization Guidance
- KO는 의미 중심으로 다듬고, 다른 언어는 자연스러운 현지 표현으로 변환.
- 직역 금지: "입맛" 등 문화권별 자연어로 변환.
- 고유명사: mimy, 캐치테이블 등은 유지.


## 5) UX Writing Agent Spec
### 목적
- KO 기준으로 모든 카피를 톤앤매너에 맞게 정리하고,
  다른 언어가 이 톤을 재현하도록 기준점을 만든다.

### 입력
- 문자열
- UI 문맥(화면/기능/상태)

### 출력
- 개선된 문자열
- 변경 이유(짧게)

### 체크리스트
- {{variable}} / <1> / <br/> 등 토큰 보존
- 의미 변화 금지
- 문장 길이 과도하게 증가하지 않음
- KO 톤 규칙 적용
- 용어집 준수

### 작업 순서
1. 문맥 파악 (온보딩/기록/추천/에러)
2. 톤 레벨 선택 (공감/정보/경고)
3. 문장 단축 및 리듬 정리
4. 용어 및 띄어쓰기 정리
5. 토큰/태그 보존 확인
