# mimy 서비스 소개서 (법무팀용)

> **작성일**: 2026-02-12
> **서비스명**: mimy (My Michelin for Me)
> **한 줄 요약**: 사용자의 미식 성향을 분석하여 개인화된 맛집 추천을 제공하는 소셜 미식 플랫폼

---

## 1. 서비스 개요

mimy는 "나만의 미슐랭"을 지향하는 맛집 추천 플랫폼입니다. 사용자가 간단한 미식 성향 테스트를 거치면, 비슷한 취향의 다른 사용자들의 평가를 기반으로 "이 맛집이 나한테 맞을 확률"을 0~100% 점수로 보여줍니다.

핵심 가치는 **개인화**입니다. 똑같은 식당이라도 사용자 A에게는 92%, 사용자 B에게는 45%로 다르게 보입니다. 별점 평균이 아니라 "나와 입맛이 비슷한 사람들이 좋아한 곳"을 찾아주는 것이 차별점입니다.

### 현재 운영 상태

- **비공개 베타**: 현재 `@catchtable.co.kr` 이메일로만 가입 가능 (사내 테스트)
- **플랫폼**: 웹 브라우저 + iOS/Android 하이브리드 앱
- **언어**: 한국어, 영어

## 2. 사용자 여정 (처음부터 끝까지)

### 2.1 앱 시작 → 가입까지

```
앱 실행 → 브랜드 소개 (4초) → 서비스 소개 슬라이드 3장 → "시작하기" 버튼
```

사용자가 "시작하기"를 누르면 아래 순서로 가입이 진행됩니다.

**① 연령 확인**
- 생년월일을 스크롤 휠로 선택
- 만 14세 미만은 가입 불가 (차단 메시지 표시)

**② 약관 동의**
- 필수 약관 4개 + 선택 약관 1개를 체크박스로 표시
- "전체 동의" 옵션 제공
- 각 약관을 누르면 전문을 읽을 수 있음
- 필수 약관을 모두 동의해야 다음 단계로 진행 가능

| 약관 | 필수 여부 |
|------|-----------|
| 서비스 이용약관 | 필수 |
| 개인정보 처리방침 | 필수 |
| 위치기반서비스 이용약관 | 필수 |
| 개인위치정보 수집 및 이용 동의 | 필수 |
| 마케팅 정보 수신 동의 | 선택 |

**③ Google 로그인**
- Google 계정으로 로그인 (현재 @catchtable.co.kr만 허용)
- 신규 사용자는 회원가입 플로우로 이동

**④ 휴대폰 인증**
- 국가 선택 후 휴대폰 번호 입력 (한국, 일본, 대만, 홍콩, 싱가포르, 태국, 미국, 호주 지원)
- SMS로 6자리 인증번호 수신, 3분 내 입력

**⑤ 프로필 설정** (3단계)
1. **사용자명 (@handle)**: 영문 소문자, 숫자, _, . 사용 가능. 실시간 중복 체크
2. **닉네임**: 서비스 내 표시 이름 (최대 30자)
3. **프로필 사진**: 카메라/갤러리에서 선택 (선택사항)

### 2.2 미식 성향 테스트 (퀴즈)

가입 직후 자동으로 미식 성향 테스트가 시작됩니다.

**방식**: 음식 관련 질문이 카드로 표시되며, 좋으면 오른쪽 스와이프 / 싫으면 왼쪽 스와이프

**분석 기준 (7개 축)**:
| 축 | 설명 | 예시 |
|----|------|------|
| Boldness | 강한 맛 선호도 | 진한 국물, 훈제 |
| Acidity | 산미 선호도 | 레몬, 식초, 발효 |
| Richness | 풍미/기름진 맛 선호도 | 크림소스, 버터 |
| Experimental | 새로운 맛 도전 성향 | 퓨전, 이색 재료 |
| Spiciness | 매운 맛 선호도 | 마라, 고추 |
| Sweetness | 단 맛 선호도 | 디저트, 달달한 양념 |
| Umami | 감칠맛 선호도 | 숙성, 발효, 해산물 |

**결과**: 7개 축의 점수 조합으로 **128개 미식 클러스터** 중 하나로 분류됩니다.

각 클러스터에는 재미있는 이름이 붙습니다:
- "상큼한 탐험가" — 산미와 감칠맛을 좋아하는 도전적 성향
- "달콤한 솜사탕" — 단순한 단맛을 선호하는 성향
- "화끈한 마라샹궈" — 강한 매운맛과 임팩트를 좋아하는 성향
- "침착한 중용자" — 모든 맛의 균형을 추구하는 성향

이 클러스터 정보가 이후 맛집 추천의 핵심 기반이 됩니다.

### 2.3 메인 서비스 이용

가입과 퀴즈가 끝나면 메인 화면으로 진입합니다. 하단에 5개 탭이 있습니다.

## 3. 주요 기능 상세

### 3.1 홈 피드

팔로우한 사용자와 추천 사용자의 리뷰/포스트가 타임라인 형태로 표시됩니다.

각 게시물에 표시되는 정보:
- 작성자 프로필 (닉네임, 사진, 미식 클러스터 뱃지)
- 맛집 이름, 카테고리
- 사진 (여러 장이면 캐로셀로 넘겨볼 수 있음)
- 텍스트 리뷰
- 좋아요 수, 댓글 수
- 작성일

사용자는 좋아요, 댓글, 공유 등의 상호작용이 가능합니다.

### 3.2 맛집 탐색 (Discovery)

지도 기반의 맛집 탐색 화면입니다.

**동작 방식**:
- 지도 위에 맛집들이 핀으로 표시됨
- 각 핀에 해당 사용자의 **예측 매칭 점수** 가 함께 표시됨
- 핀을 누르면 맛집 카드가 나타남 (이름, 카테고리, 주소, 매칭 점수, 최근 리뷰)

**필터 기능**:
- 카테고리 필터 칩: 한식, 일식, 중식, 이탈리안, 퓨전, 스테이크 등
- "높은 매칭" 필터: 사용자의 매칭 점수가 높은 곳만 표시
- 텍스트 검색

**위치 정보 사용**: 사용자의 현재 위치를 기준으로 주변 맛집을 표시하고, 거리를 계산합니다.

### 3.3 맛집 매칭 점수 (핵심 알고리즘)

mimy의 핵심 기능입니다. 법무팀이 이해해야 할 부분을 비기술적으로 설명합니다.

**"이 맛집이 나한테 맞을 확률 87%"는 어떻게 나오는가?**

1. 나와 비슷한 미식 클러스터의 사용자들을 찾음
2. 그 사용자들이 이 맛집을 어떻게 평가했는지 수집
3. 나와 취향이 더 비슷할수록 그 사람의 평가에 더 높은 가중치 부여
4. 이를 종합하여 0~100% 점수 산출

**신뢰성 기준**:
- 100개 이상 맛집을 평가한 사용자의 데이터만 사용
- 최소 3명 이상의 신뢰할 수 있는 리뷰어가 있어야 점수 표시

**개인정보 관점**:
- 매칭 점수는 집계 데이터 기반으로 계산됨 (개별 사용자 평가를 직접 노출하지 않음)
- 점수는 요청 시 실시간 계산되며, 사용자별로 다르게 표시됨

### 3.4 리뷰 작성

사용자가 방문한 맛집에 대한 리뷰를 작성하는 플로우입니다.

**Step 1: 맛집 선택 & 만족도 평가**
1. 맛집 이름으로 검색 (Google Places 데이터 연동)
2. 만족도 선택: Good 😊 / OK 😐 / Bad 😞
3. (선택) 기존 랭킹과 비교하여 순위 결정
   - 예: "지난번 평가한 'A식당'보다 맛있었나요?"

**Step 2: 리뷰 내용 작성**
- 사진 1~5장 업로드
- 텍스트 리뷰 작성
- 방문 날짜 입력
- 동행자 태그 (다른 사용자 멘션)

작성이 완료되면 피드에 게시되고, 팔로워들에게 노출됩니다.

### 3.5 빠른 기록 (릴레이)

리뷰를 쓰지 않고 맛집 만족도만 빠르게 기록하는 기능입니다.

**사용 방식**: Tinder 스타일의 카드 스와이프
- 맛집 카드가 한 장씩 표시됨
- 오른쪽 스와이프 → Good (맛있었어요)
- 왼쪽 스와이프 → Bad (별로였어요)
- 위로 스와이프 → OK (괜찮았어요)
- 아래 버튼 → 안 가봤어요 (건너뛰기)

**비교 프롬프트**: 5개마다 "A식당과 B식당 중 어디가 더 좋았나요?" 비교 질문이 등장

**마일스톤**: 30개를 평가하면 "랭킹을 정리하시겠습니까?" 모달이 등장하여 일괄 저장 가능

### 3.6 개인 맛집 랭킹

사용자가 평가한 모든 맛집이 순위대로 정리됩니다.

**구조**:
```
Good (맛있었어요)
  1위: A식당
  2위: B식당
  3위: C식당
OK (괜찮았어요)
  4위: D식당
  5위: E식당
Bad (별로였어요)
  6위: F식당
```

- 순위는 Good → OK → Bad 순서로 연속됨 (티어별 독립 순위가 아님)
- 드래그 앤 드롭으로 순서 변경 가능
- 이 랭킹 데이터가 다른 사용자의 매칭 점수 계산에 활용됨

### 3.7 리더보드

사용자들의 활동을 기반으로 한 순위표입니다.

**3가지 보기**:
| 보기 | 설명 |
|------|------|
| 전체 | 모든 사용자 순위 |
| 소속 | 같은 회사/학교 소속 사용자 순위 |
| 동네 | 같은 동네에 등록된 사용자 순위 |

### 3.8 소셜 기능

**팔로우/팔로워**: 다른 사용자를 팔로우하면 피드에 해당 사용자의 리뷰가 표시됩니다.

**좋아요**: 리뷰/포스트에 좋아요를 누를 수 있습니다.

**댓글**: 리뷰/포스트에 댓글을 작성할 수 있으며, 대댓글(답글)도 지원됩니다.

**프로필 조회**: 다른 사용자의 프로필을 누르면 그 사용자의 리뷰 목록, 팔로워 수, 미식 클러스터, 나와의 취향 일치도를 확인할 수 있습니다.

### 3.9 소속 등록

사용자가 회사/학교 또는 동네를 등록하여 소속 기반 리더보드에 참여할 수 있습니다.

**회사/학교 등록**:
1. 등록 가능한 그룹 목록에서 선택
2. 해당 조직의 이메일 주소 입력 (예: user@catchtable.co.kr)
3. 인증 코드가 이메일로 발송 (6자리, 5분 내 입력, 최대 5회 시도)
4. 인증 완료 시 소속 등록
5. 소속 변경은 30일에 1회 가능

**동네 등록**:
1. 위치 권한 허용
2. GPS로 현재 위치 파악 → 역지오코딩으로 행정구역 변환
3. "OO동으로 등록하시겠습니까?" 확인
4. 동네 변경은 30일에 1회 가능

## 4. 사용자 데이터 흐름 요약

### 4.1 가입 시 수집하는 정보

| 항목 | 수집 시점 | 용도 |
|------|-----------|------|
| 생년월일 | 연령 확인 단계 | 14세 미만 가입 제한 |
| Google 계정 (이메일, 이름) | 로그인 | 본인 확인, 계정 식별 |
| 휴대폰 번호 | SMS 인증 | 본인 확인, 중복 가입 방지 |
| 사용자명 (@handle) | 프로필 설정 | 서비스 내 고유 식별자 |
| 닉네임 | 프로필 설정 | 서비스 내 표시 이름 |
| 프로필 사진 | 프로필 설정 (선택) | 프로필 표시 |

### 4.2 서비스 이용 중 생성/수집되는 정보

| 항목 | 생성 시점 | 용도 |
|------|-----------|------|
| 미식 성향 (7축 점수, 클러스터) | 퀴즈 응답 | 맛집 매칭 점수 계산의 핵심 |
| 맛집 만족도 평가 (Good/OK/Bad) | 리뷰 작성, 릴레이 | 개인 랭킹, 다른 사용자 매칭 점수 계산에 활용 |
| 맛집 순위 (랭킹) | 리뷰 작성, 릴레이 | 개인 랭킹 관리, 매칭 점수 계산 |
| 리뷰 텍스트 | 리뷰 작성 | 피드 게시, 맛집 상세 페이지 표시 |
| 사진 | 리뷰 작성 | 피드 게시 (클라우드 저장) |
| 좋아요, 댓글 | 소셜 활동 | 피드 노출, 알림 |
| 팔로우 관계 | 소셜 활동 | 피드 구성, 프로필 표시 |
| 현재 위치 (GPS) | 지도 탐색, 동네 등록 | 주변 맛집 표시, 거리 계산, 동네 파악 |
| 동네 정보 (행정구역명) | 동네 등록 | 동네 리더보드, 동네 기반 추천 |
| 소속 정보 (회사/학교) | 이메일 인증 | 소속 리더보드 |
| 못 먹는 음식 목록 | 설정 | 추천 필터링 |
| 밸런스 게임 투표 | 게임 참여 | 커뮤니티 콘텐츠 |

### 4.3 자동 수집 정보

| 항목 | 용도 |
|------|------|
| 서비스 이용 기록 | 서비스 개선, 통계 |
| 접속 로그 | 보안, 부정이용 방지 |
| 쿠키 | 인증 토큰 유지 (JWT) |
| 기기 정보 (OS, 기기 식별자) | 서비스 최적화 |

### 4.4 외부 서비스 데이터 연동

| 외부 서비스 | 연동 내용 | 전송되는 데이터 |
|-------------|-----------|----------------|

| Google Places API | 맛집 정보 검색 | 검색어 (mimy → Google) |
| MapTiler | 지도 표시, 역지오코딩 | GPS 좌표 (mimy → MapTiler) |

| Resend | 이메일 발송 | 수신 이메일 주소, 인증 코드 (mimy → Resend) |

| 메뉴 | 기능 |

## 6. 약관 동의 기록 관리

약관 동의 내역은 데이터베이스에 아래 정보와 함께 기록됩니다.

| 기록 항목 | 설명 |
|-----------|------|
| 사용자 ID | 동의한 사용자 식별 |
| 약관 ID | 어떤 약관에 동의했는지 |
| 약관 버전 | 동의 시점의 약관 버전 |
| 동의 여부 | 동의/미동의 |
| 동의 일시 | 동의한 정확한 시각 |
| IP 주소 | 동의 시 접속 IP |
| User-Agent | 동의 시 브라우저/기기 정보 |

동일한 사용자가 같은 약관의 같은 버전에 중복 동의할 수 없도록 제약조건이 설정되어 있습니다.
